## 目录负责规则

| 目录 | 负责人 | 说明 |
|---|---|---|
| `00_project_docs/` | A | 项目规范文档，其他成员只读 |
| `01_data/` | A | 数据清洗、特征工程、EDA |
| `02_model/` | A | 模型训练、模型评估、模型导出 |
| `03_backend/` | B | Flask、MySQL、API、模型调用 |
| `04_frontend/` | C | 页面、图表、交互 |
| `05_bigdata/` | B，A 协助 | Flume/Kafka/Storm 配置和模拟流程 |
| `06_reports/` | 全员 | 报告、PPT、截图 |
| `07_ai_prompts/` | A | 给各成员复制到 AI 的提示词 |

## 禁止事项

1. B/C 不要改 `字段规范.md`。
2. B/C 不要私自改 `接口文档.md`。
3. C 不要直接读取 CSV，前端必须调用后端 API。
4. B 不要重新训练模型，只调用 A 提供的模型文件。
5. 任何人不要覆盖其他成员负责目录下的文件。
