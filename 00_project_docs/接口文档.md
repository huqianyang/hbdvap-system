# 接口文档

本文档用于统一后端 B 和前端 C 的接口契约。B 必须按本文档开发 Flask API，C 必须按本文档调用接口。

## 1. 通用响应格式

所有接口统一返回 JSON：

```json
{
  "success": true,
  "message": "请求成功",
  "data": {}
}
```

失败时：

```json
{
  "success": false,
  "message": "错误原因",
  "data": null
}
```

## 2. 接口列表

| 接口 | 方法 | 用途 | 负责人 |
|---|---|---|---|
| `/api/dashboard` | GET | 首页看板汇总数据 | B |
| `/api/predict` | POST | 预订取消预测 | B |
| `/api/analysis/cancel-rate` | GET | 取消率趋势分析 | B |
| `/api/analysis/hotel-type` | GET | 酒店类型分析 | B |
| `/api/analysis/country` | GET | 国家/地区分析 | B |
| `/api/analysis/customer-type` | GET | 客户类型分析 | B |
| `/api/bigdata/status` | GET | 大数据组件状态说明 | B |

## 3. 首页看板接口

### GET `/api/dashboard`

用途：返回首页核心指标。

### 响应示例

```json
{
  "success": true,
  "message": "请求成功",
  "data": {
    "total_bookings": 119390,
    "canceled_bookings": 44224,
    "cancel_rate": 0.3704,
    "avg_adr": 101.83,
    "city_hotel_count": 79330,
    "resort_hotel_count": 40060,
    "top_country": "PRT"
  }
}
```

## 4. 预测接口

### POST `/api/predict`

用途：接收前端输入的一条预订信息，调用模型预测是否取消。

### 请求示例

```json
{
  "hotel": "City Hotel",
  "lead_time": 120,
  "arrival_date_month": "July",
  "stays_in_weekend_nights": 1,
  "stays_in_week_nights": 3,
  "adults": 2,
  "children": 0,
  "babies": 0,
  "meal": "BB",
  "country": "PRT",
  "market_segment": "Online TA",
  "distribution_channel": "TA/TO",
  "is_repeated_guest": 0,
  "previous_cancellations": 0,
  "previous_bookings_not_canceled": 0,
  "reserved_room_type": "A",
  "booking_changes": 0,
  "deposit_type": "No Deposit",
  "days_in_waiting_list": 0,
  "customer_type": "Transient",
  "adr": 90.0,
  "required_car_parking_spaces": 0,
  "total_of_special_requests": 1
}
```

### 响应示例

```json
{
  "success": true,
  "message": "预测成功",
  "data": {
    "is_canceled_pred": 1,
    "cancel_probability": 0.82,
    "risk_level": "高风险",
    "suggestion": "建议酒店提前确认客户入住意愿"
  }
}
```

## 5. 取消率趋势接口

### GET `/api/analysis/cancel-rate`

用途：按月份返回取消率趋势，供前端绘制折线图。

### 响应示例

```json
{
  "success": true,
  "message": "请求成功",
  "data": {
    "x_axis": ["January", "February", "March", "April"],
    "cancel_rate": [0.31, 0.34, 0.36, 0.38],
    "booking_count": [5000, 6200, 7100, 8300]
  }
}
```

## 6. 酒店类型分析接口

## 9. 大数据组件状态接口

### GET `/api/bigdata/status`

用途：给前端展示 Flume、Kafka、Storm 的数据流处理状态或模拟状态。

### 响应示例

```json
{
  "success": true,
  "message": "请求成功",
  "data": {
    "flume": "已配置，负责采集CSV数据",
    "kafka": "已配置，负责消息队列缓冲",
    "storm": "已配置，负责实时清洗、预测和统计",
    "pipeline": "CSV -> Flume -> Kafka -> Storm -> MySQL -> Flask -> Web"
  }
}
```

## 10. 前后端联调规则

1. B 先用 Postman 或浏览器测试所有 GET 接口。
2. B 用固定 JSON 测试 `/api/predict`。
3. C 只调用本文档中的接口，不要自己改接口地址。
4. 如果接口返回格式需要修改，必须先改本文档，再同步给 A/B/C。
