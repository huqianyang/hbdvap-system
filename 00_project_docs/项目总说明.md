# 酒店预订数据可视化分析与预测系统 - 项目总说明

## 1. 项目基本信息

**项目名称**：酒店预订数据可视化分析与预测系统  
**英文名称**：Hotel Booking Data Visualization Analysis & Prediction System  
**项目简称**：HBDVAP System  
**课程类型**：《数据可视化》期末课程设计  
**数据集**：Kaggle Hotel Booking Demand（酒店预订需求数据集）  
**数据规模**：约 119,390 条记录，32 个原始字段  

## 2. 项目背景

酒店行业中，客户预订后取消订单是常见问题。大量取消会影响酒店房间安排、收益管理和人员调度。本项目基于酒店历史预订数据，分析影响取消的因素，并训练机器学习模型预测新预订是否可能取消。

同时，课程要求使用大数据相关组件，因此本项目设计为一个包含数据采集、消息队列、实时处理、数据库存储、后端接口和前端可视化的完整系统。

## 3. 核心功能

1. **预订取消预测**
   - 输入一条酒店预订信息
   - 模型预测该订单是否可能取消
   - 输出取消概率、风险等级和建议

2. **数据可视化看板**
   - 展示整体预订量、取消率、平均房价等指标
   - 展示不同酒店类型、月份、国家/地区、客户类型下的取消情况

3. **多维度分析**
   - 按时间分析取消率趋势
   - 按酒店类型分析取消差异
   - 按客户类型分析客户画像
   - 按国家/地区分析订单来源

4. **大数据流处理架构**
   - Flume：模拟采集 CSV 预订数据
   - Kafka：作为消息队列缓冲预订数据
   - Storm：实时处理数据，完成清洗、预测和统计
   - MySQL：存储清洗数据、预测结果和统计结果
