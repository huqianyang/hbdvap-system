# 字段规范

本文档用于统一成员 A、B、C 在数据处理、后端接口和前端展示中使用的字段名称，避免字段不一致导致联调失败。

## 1. 数据集说明

数据集名称：Hotel Booking Demand  
数据文件：`hotel_bookings.csv`  
原始记录数：约 119,390 条  
原始字段数：32 个  
预测目标字段：`is_canceled`

## 2. 目标字段

| 字段名 | 含义 | 类型 | 说明 |
|---|---|---|---|
| `is_canceled` | 是否取消预订 | int | 0=未取消，1=已取消，是模型预测目标 |

## 4. 模型输入字段建议

为了避免标签泄露，模型训练时不要使用 `reservation_status` 和 `reservation_status_date`。

建议第一版模型使用以下字段：

```text
hotel
lead_time
arrival_date_month
stays_in_weekend_nights
stays_in_week_nights
adults
children
babies
meal
country
market_segment
distribution_channel
is_repeated_guest
previous_cancellations
previous_bookings_not_canceled
reserved_room_type
booking_changes
deposit_type
days_in_waiting_list
customer_type
adr
required_car_parking_spaces
total_of_special_requests
```

## 6. 模型输出字段

| 字段名 | 含义 | 类型 | 示例 |
|---|---|---|---|
| `is_canceled_pred` | 预测是否取消 | int | `1` |
| `cancel_probability` | 取消概率 | float | `0.82` |
| `risk_level` | 风险等级 | string | `高风险` |
| `suggestion` | 建议 | string | `建议酒店提前确认客户入住意愿` |

## 7. 风险等级规则

| 取消概率 | 风险等级 |
|---|---|
| `< 0.30` | 低风险 |
| `0.30 - 0.70` | 中风险 |
| `> 0.70` | 高风险 |

## 8. 成员协作要求

1. A 负责维护本字段规范。
2. B 写后端时必须使用本文件字段名。
3. C 写前端表单和图表时必须使用接口文档返回字段，不要自己造字段。
4. 任何字段新增、删除、改名，都必须先同步给全员。
