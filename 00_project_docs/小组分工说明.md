# 小组分工说明

本文档用于明确 A/B/C 三名成员的职责边界，避免互相覆盖文件、重复开发或接口冲突。

## 1. 总体原则

1. A 是组长，负责项目规则、字段、接口和最终整合。
2. B 负责后端、大数据和数据库，不负责训练模型。
3. C 负责前端和可视化，不直接读取 CSV，不直接训练模型。
4. 所有人都可以使用 AI 工具，但必须使用统一提示词和统一文档。
5. 字段、接口、目录不能私自修改。

## 2. 成员 A：组长 + 数据算法工程师

### 负责目录

```text
00_project_docs/
01_data/
02_model/
07_ai_prompts/
README.md
requirements.txt
```

### 核心任务

1. 整理项目总说明、目录结构、字段规范、接口文档。
2. 读取并分析 `hotel_bookings.csv`。
3. 完成数据清洗和特征工程。
4. 训练酒店预订取消预测模型。
5. 导出 `cancellation_model.pkl`。
6. 提供 `predict_demo.py` 给 B 调用模型。
7. 检查 B/C 是否遵守字段和接口。
8. 汇总报告、PPT 和最终项目压缩包。

### 不能拖延的任务

模型文件必须尽早完成，否则 B 的预测接口无法真正联调，C 的预测页面也无法展示真实结果。

## 3. 成员 B：后端 + 数据库 + 大数据工程师

### 负责目录

```text
03_backend/
05_bigdata/
```

### 核心任务

1. 搭建 Flask 后端。
2. 设计 MySQL 数据库表。
3. 将 A 清洗后的数据导入 MySQL。
4. 按 `接口文档.md` 编写 API。
5. 调用 A 提供的 `cancellation_model.pkl`。
6. 编写大数据模块配置和模拟脚本。
7. 提供接口测试结果给 A 和 C。

### 必须完成的接口

```text
GET  /api/dashboard
POST /api/predict
GET  /api/analysis/cancel-rate
GET  /api/analysis/hotel-type
GET  /api/analysis/country
GET  /api/analysis/customer-type
GET  /api/bigdata/status
```

### 禁止事项

1. 不要修改字段规范。
2. 不要自己设计新的接口格式。
3. 不要重新训练模型。
4. 不要让前端直接读数据库。
5. 不要把接口返回格式写得和接口文档不一致。

## 4. 成员 C：前端 + 可视化工程师

### 负责目录

```text
04_frontend/
```

### 核心任务

1. 设计首页和整体页面风格。
2. 制作数据看板页面。
3. 制作预测输入页面。
4. 制作多维分析页面。
5. 使用 jQuery 调用 B 的 Flask API。
6. 使用 ECharts/PyECharts 绘制图表。
7. 展示大数据架构流程。

### 必须完成的页面

```text
首页 index.html
数据看板 dashboard.html
预测页面 prediction.html
多维分析 analysis.html
公共模板 base.html
```

### 必须展示的内容

1. 总预订量、取消量、取消率、平均房价。
2. 月度取消率趋势折线图。
3. 酒店类型取消率对比图。
4. 国家/地区订单分布图。
5. 客户类型分布图。
6. 预测输入表单和预测结果卡片。
7. Flume → Kafka → Storm → MySQL → Flask → Web 数据流说明。

### 禁止事项

1. 不要直接读取 CSV。
2. 不要写死假数据作为最终结果。
3. 不要改接口地址和返回字段。
4. 不要自己新增后端接口。

## 5. 联调方式

### A 和 B 联调

1. A 提供模型文件和调用示例。
2. B 在 Flask 中加载模型。
3. B 用固定 JSON 测试 `/api/predict`。
4. A 检查预测结果是否合理。

### B 和 C 联调

1. B 提供接口地址和示例返回。
2. C 用 jQuery 请求接口。
3. C 渲染图表和预测结果。
4. B/C 一起排查接口路径、跨域、字段名问题。

### 全员联调

测试完整流程：

```text
打开前端页面
  ↓
查看首页看板
  ↓
进入预测页面
  ↓
输入预订信息
  ↓
点击预测
  ↓
后端调用模型
  ↓
前端展示取消概率和风险等级
```

```text

统一字段
统一接口
统一提示词
```
