# Python 数据可视化课程设计要求

## 第一阶段：选题与数据准备

### 全员参与

这一步决定了项目的天花板和可行性。

1. **确定课题方向**：结合你们的兴趣和专业，寻找具有实际业务价值的场景。

2. **寻找并评估数据集**：
   - **数据源**：优先从 Kaggle、和鲸社区、百度飞桨、天池、政府公开数据平台等等找现成的开放数据集。如果找不到合适的，再考虑使用 Python 爬虫（如 Requests、Selenium）从网站抓取。
   - **评估难度**：数据量是否已经可以跑机器学习？字段是否丰富（包含数值型、类别型）？是否有时间序列（适合趋势预测）？

3. **最终技术选型确认**：
   - **后端**：Python + Flask（轻量灵活，适合快速开发 API）
   - **前端**：HTML + Tailwind + jQuery
   - **机器学习**：Scikit-learn（传统机器学习算法）、Pandas & NumPy（数据处理）
   - **大数据框架**：如果数据量在百万以下，Pandas 足够；如果数据量很大，可引入 PySpark 进行分布式处理
   - **数据库**：MySQL（存储结构化业务数据）+Redis
   - **大数据框架**：Flume+Kafka+storm

## 第二阶段：小组分工（并行开发）

三人小组建议采用"全栈互补"的分工模式，每个人有主攻方向，但互相了解彼此口。

### 角色分工表格

| 角色 | 核心职责 | 关键技术栈 |
|------|----------|------------|
| **成员A：数据与算法工程师** | 负责数据清洗、特征工程、机器学习模型（预测/分类/聚类）的训练与评估，导出模型文件。 | Python, Pandas, Scikit-learn, Jupyter |
| **成员B：后端与大数据工程师** | 搭建 Flask 后端，设计 MySQL 数据库，编写数据入库脚本，开发 RESTful API 接口供前端调用。 | Flask, PyECharts, MySQL, SQLAlchemy, PySpark(可选) |
| **成员C：前端与可视化工程师** | 负责 Web 页面布局（HTML/CSS/JS），使用 PyECharts 绘制交互式图表，对接后端 API 渲染数据。 | HTML, Tailwind, jQuery |

## 第三阶段：核心开发步骤

### 步骤1：数据工程与建模（成员A主导）

- **数据清洗**：处理缺失值、异常值，进行数据类型转换。
- **特征工程**：对类别特征进行编码（如 One-Hot）、提取时间特征，进行数据归一化。
- **模型训练**：根据课题选择合适的回归/分类/聚类算法（如 K-Means 聚类、XGBoost），做用户分群可用 K-Means 聚类。
- **模型保存**：使用 joblib 或 pickle 将训练好的模型保存为 .pkl 文件，方便后端调用。

### 步骤2：后端开发与接口封装（成员B主导）

- **数据库设计**：根据数据字段设计 MySQL 表结构（如商品表、用户表、预测结果表）。
- **数据入库**：编写 Python 脚本将清洗好的历史数据导入 MySQL。
- **API 开发**：
  - 开发数据查询接口（如 `/api/get_sales_data`），返回 JSON 格式数据供前端调用。

## 前端开发方案

针对你的项目特点（Flask 后端 + PyECharts 可视化 + 机器学习），我为你梳理了以下三种最主流且可行的前端技术栈：

### 方案一：高性价比流（强烈推荐）

**技术栈**：Flask + HTML + Tailwind + jQuery

Flask + HTML + Tailwind CSS + jQuery 的组合，完美避开了现代前端工程化（如 Vue/React 的打包、路由、状态管理）的复杂门槛，同时又能实现足够美观、交互足够流畅。

#### Flask（后端中枢）
- 负责接收前端发来的请求，调用你写好的数据处理脚本、机器学习模型（如预测/分类/聚类、线性回归等）进行预测，并将处理后的数据通过 Jinja2 模板引擎渲染到 HTML 页面中。

#### HTML + Jinja2（页面骨架）
- 作为 Flask 的模板，负责搭建页面的基础结构，你可以在 HTML 中直接使用 `{{ data }}` 或 `{% for item in items %}` 等 Jinja2 语法，将后端数据直接嵌入到页面里。

#### Tailwind CSS（界面美化）
- 负责页面的"精装修"，你不需要写一行传统的 CSS 代码，直接在 HTML 标签的 class 属性中写如 `bg-blue-500 text-white p-4 rounded` 这样的工具类，就能快速构建出优美现代化感、完全响应式的数据大屏和单页面。

#### jQuery（交互与图表驱动）
- 负责页面动态交互方法，在你的项目中，jQuery 主要有两个核心作用：
  1. **发送异步请求**：比如用户点击"开始预测"按钮时，用 `$.ajax` 或 `$.get` 向后端 Flask 请求预测结果，实现局部刷新，不用重新加载整个网页。
  2. **驱动 PyECharts 渲染**：这是最关键的一点，你可以将后端传来的 PyECharts 图表配置（通过 `chart.dump_options()` 转成的 JSON 数据），在 jQuery 中通过 ECharts 的 JS 接口（`echarts.init` 和 `setOption`）直接渲染到页面的 `<div>` 容器中。

### 方案二：极速开发流

**技术栈**：Streamlit（纯 Python 框架）+ PyECharts

如果你更想将精力集中在数据清洗、清洗、AI建模等核心业务逻辑上，且希望快速交付一个可交互的网页系统，这是目前性价比最高的选择。

#### 核心逻辑
- 完全摆脱 HTML/CSS/JS 的编写，Streamlit 允许你只用 Python 脚本就能生成包含会话功能、按钮、数据表格和图表的交互式网页。

#### 为什么适合你
1. **与 PyECharts 完美整合**：Streamlit 提供了 `streamlit-echarts` 等组件，可以直接将你在 Python 中生成的 PyECharts 图表嵌入到网页上，无需任何前端配置。
2. **开发速度极快**：不需要配置 Webpack、Vite 等前端工程化工具，不需要写引入路由，你只需要编写 Python 脚本，非常适合数据分析师和数据科学家项目。
3. **避开短板**：如果你不擅长 JS 生态，这个方案彻底让你用 Python 搞定从后端到前端的所有事情。

---

### 方案三：经典前后端分离流（主流企业级）

**技术栈**：Vue.js / React + Flask (RESTful API) + ECharts.js

如果你希望通过这个项目去学习更现代 Web 全栈开发流程，并且对前端的定制化程度、交互流畅度有极高要求，那么经典的"前后端分离方案"是最佳实践。

#### 核心逻辑
- **前端**：使用 Vue 或 React 搭建单页应用（SPA），通过 Axios 等工具调用后端的 RESTful API 接口，获取数据后用 ECharts.js 渲染图表。
