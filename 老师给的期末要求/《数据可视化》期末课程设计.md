# 《数据可视化》期末课程设计

## 1. 项目背景与意义

本课程设计旨在让学生通过构建一个完整的 Web 应用，掌握从数据采集、清洗、人工智能建模到可视化展示的全流程技能。项目要求整合大数据处理框架，结合 **Flask** 后端框架、**PyECharts** 可视化库以及 **机器学习算法**，实现一个具有实际业务价值的“实时评估与分析预测系统”，具体**题目自拟**。目标为巩固学生的编程基础，深化其对“数据驱动决策”和"AI 赋能可视化”的理解。

## 2. 设计目标

1. **技术栈掌握**：熟练掌握大数据框架，如Flume、Kafka以及storm； Flask Web 框架的路由、模板渲染及 API 设计；精通 PyECharts 的各种图表配置与交互效果。
2. **AI 应用能力**：能够使用 Python 机器学习库（如 Scikit-learn, XGBoost 等）构建回归预测模型与聚类模型，并将模型集成到 Web 应用中。
3. **全栈工程思维**：理解前后端数据交互（JSON/AJAX）、数据库设计（SQLite/MySQL）及系统部署的基本流程。
4. **数据叙事能力**：通过可视化图表有效地传达数据背后的规律，而非单纯地堆砌图表。

## 3. 系统架构设计

系统采用 **B/S (Browser/Server)** 架构，分为四层：

1. **数据层 (Data Layer)**：
   - 数据采集与缓冲：Flume+Kafka。
   - 数据处理框架：实时数据处理框架storm。
   - 数据存储：redis+mysql。
2. **算法层 (AI Layer)**：
   - 负责数据预处理（缺失值填充、独热编码、标准化）。
   - 负责模型训练、评估与保存。
   - 提供预测接口（输入特征 -> 输出估价）。
3. **服务层 (Backend Layer - Flask)**：
   - 处理 HTTP 请求。
   - 调用算法层进行实时预测。
   - 从数据库聚合数据，生成 PyECharts 所需的 JSON 配置项。
4. **表现层 (Frontend Layer)**：
   - HTML/CSS/JS 基础页面。
   - 嵌入 PyECharts 生成的交互式图表。
   - 用户输入表单。

## 4.数据来源与处理建议

- **数据来源**：
  - 推荐公开数据集（如 Kaggle 上的 House Price 数据集）。
  - 也可以编写爬虫（需注意 Robots 协议与法律风险）。
- **数据规模**：有效数据条目建议不少于 50000 条，以保证模型训练效果。
- **字段要求**：至少包含 8 个以上特征字段（其中必须包含数值型和分类型）。

## 5.实施计划（建议周期：4 周）

- **第 1 周：需求分析与数据准备**
  - 确定数据源。
  - 完成数据清洗与探索性分析（EDA）。
  - 搭建 Flask 基础框架。
- **第 2 周：模型构建与训练**
  - 特征工程。
  - 模型选型、训练、调参。
  - 保存模型文件，编写预测接口。
- **第 3 周：可视化开发与集成**
  - 使用 PyECharts 绘制核心图表。
  - 前端页面布局与美化。
  - Flask 路由与图表数据对接。
- **第 4 周：架构整合，测试、优化与文档**
  - 与flume、kafka等架构进行整合
  - 系统联调，修复 Bug。
  - 撰写课程设计报告。
  - 准备答辩演示。

## 6.考核评价标准

| 维度          | 权重  | 评分细则                                |
| ----------- | --- | ----------------------------------- |
| **功能完整性**   | 30% | 功能模块是否全部实现，系统能否正常运行无报错。             |
| **可视化效果**   | 25% | PyECharts 图表类型丰富度、交互性、美观度、布局合理性。    |
| **AI 模型质量** | 15% | 模型选择合理性、评估指标（R²等）是否达标、是否有过拟合/欠拟合分析。 |
| **代码规范**    | 5%  | 代码注释清晰、目录结构规范。                      |
| **报告与答辩**   | 25% | 文档逻辑清晰、问题分析深入、答辩演示流畅。               |

## 7.提交物要求

1. **源代码**：完整的工程文件夹（包含 `requirements.txt`）。
2. **课程设计报告**：word 格式，包含背景介绍，系统架构图以及说明、模型评估结果、核心代码片段、可视化截图及心得体会。
