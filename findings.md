# 发现记录

## 已确认内容
- 项目主题适合酒店预订取消预测与可视化分析
- 数据集规模满足课程要求，字段也足够丰富
- 课程要求明确提到需要使用 Flume、Kafka、Storm
- 当前项目目录已有 `README.md`、`progress.md`、课程要求文档
- 当前项目级 `.claude/settings.json` 已恢复为完整 29 个 hook 结构，并统一为纯文本输出风格

## 关键结论
- 项目不适合直接做成单纯的静态展示，而应围绕“数据采集、处理、预测、展示”形成完整流程
- 前端方案以 Flask + HTML + Tailwind + jQuery 为主最稳妥
- 课程要求里的大数据框架应作为系统架构亮点，不建议弱化

## 启动包阶段结论
- 对当前小组来说，最稳妥的协作方式不是让 B/C 自由问 AI，而是由组长先提供统一启动包。
- 启动包应包含项目说明、目录结构、字段规范、接口文档、数据流架构、开发时间表、小组分工和 AI 提示词。
- Flume、Kafka、Storm 不能只写成可选模块；即使先用 Python 脚本模拟，也要在目录、文档、报告中作为正式架构部分体现。
- B/C 可以使用不同 AI 工具，但必须统一目录、字段、接口和提示词。
- 字段规范和接口文档是协作核心，后续任何字段或接口改动都应先更新文档再改代码。

## 需要继续留意的点
- 用户对过程文件是否纳入本项目记录有明确偏好，需要尊重
- 后续若继续修改文件，应同步记录进度与原因
- 若继续生成正式文档，需要先统一目录结构与文档口径
